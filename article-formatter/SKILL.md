# Markdown to Word Converter

Convert Markdown articles to Word documents while preserving structure and formatting.

## When to Use

- User has a Markdown file and wants to convert it to Word format
- Generated an article using article-generator skill and needs .docx output
- Need to edit the article in Microsoft Word or WPS
- Want to share the article as a Word document
- Need to print the article with proper formatting

## Quick Start

### Basic Usage

```bash
# Convert Markdown to Word (auto-generate output filename)
python scripts/md_to_word.py article.md

# Specify output filename
python scripts/md_to_word.py article.md -o "文章标题.docx"

# Insert images from directory
python scripts/md_to_word.py article.md -i ./images -o "文章.docx"

# Full example with images
python scripts/md_to_word.py article.md -i ./images -o "文章.docx"
```

### Python Usage

```python
from scripts.md_to_word import convert_markdown_to_word

result = convert_markdown_to_word(
    input_file="article.md",
    output_file="output.docx",
    title="文章标题"
)

if result['success']:
    print(f"Converted to: {result['output_path']}")
    print(f"Elements: {result['elements_count']}")
```

## Supported Markdown Features

### Headings

```markdown
# Heading 1 (Title)
## Heading 2 (Section)
```

**Note**: All headings including the first H1 are preserved in the document.

### Bold Text

```markdown
**Bold text** or __Bold text__
```

### Lists

```markdown
- Item 1
- Item 2
- Item 3
```

### Tables

```markdown
| Header 1 | Header 2 |
|----------|----------|
| Cell 1   | Cell 2   |
| Cell 3   | Cell 4   |
```

Converted to Word table with proper formatting.

### Image Placeholders

```markdown
[[IMG: Image description here]]
```

**With `-i` option**: Inserts actual image  
**Without `-i` option**: Shows placeholder text

### Horizontal Rules

```markdown
---
```

### Auto-removed Sections

以下部分会自动从Word文档中移除：

- `*本文基于公开资料整理，仅供参考*` - 免责声明
- `**配图说明：**` 或 `**配图清单**` 及其后续所有内容 - 配图说明章节
- `---` 分隔线后的配图列表（如 `1. [[IMG:...]] - 描述`）

**注意**：配图说明章节（通常在文章末尾）会被完整移除，不会出现在Word文档中。

## Output Format

### Font Settings

- **Font**: Microsoft YaHei for Chinese text
- **Size**: 12pt for body text
- **Heading 1**: 18pt, bold
- **Heading 2**: 17pt, bold, italic, white text on black background, centered
- **Image placeholders**: 10pt, gray, italic

### Heading 2 Style (Section Titles)

章节标题（`## 标题`）采用特殊样式：

- **字体大小**: 17pt（原21pt的0.8倍）
- **字体颜色**: 白色
- **背景色**: 黑色（包括前后空格）
- **对齐方式**: 居中
- **字体样式**: 斜体
- **前后空格**: 标题文字前后各加 **1 个**空格（空格也有黑色背景）
- **上方空行**: 标题前自动添加空行
- **下方空行**: 标题后自动添加空行

示例效果：
```

[空行]
        [   章节标题文字   ]

[空行]
```

### Document Structure

- Title page (if specified)
- Headings with proper hierarchy
- Body text with inline formatting
- Lists with bullet points
- Image placeholder annotations
- Page breaks between major sections

## Script Reference

### `scripts/md_to_word.py`

**Purpose**: Convert Markdown files to Word documents

**Requirements**:
- Python 3.8+
- python-docx: `pip install python-docx`

**Arguments**:
- `input` (required): Input Markdown file path
- `-o, --output`: Output Word file path (optional, auto-generated if not specified)
- `-t, --title`: Document title (optional)

**Returns**:
```python
{
    'success': True/False,
    'output_path': 'path/to/output.docx',
    'elements_count': 25,  # Number of elements processed
    'error': 'error message'  # Only if success is False
}
```

## Integration with article-generator

This skill is designed to work with articles generated by `article-generator`:

```bash
# Step 1: Generate article
python article-generator/scripts/generate_article.py "主题" --output article.md

# Step 2: Convert to Word
python article-formatter/scripts/md_to_word.py article.md -o "文章.docx"
```

## Limitations

- Tables are not yet supported
- Links are converted to plain text
- Code blocks are converted to plain text
- Complex nested formatting may not be preserved
- Images are not embedded (only placeholders)
