# Article Shorter - 文章精简器

将长文章自动压缩成适合视频生成器使用的 `content.json`。

## 使用方法

用户直接发送文章内容，模型直接输出 JSON，无需运行任何脚本。

## 输出格式

```json
{
  "main_title": "大标题（≤10字）",
  "sub_title": "副标题（≤12字）",
  "text_sections": [
    "第一段（40-50字）",
    "第二段（40-50字）",
    "第三段（40-50字）"
  ],
  "short_article": "300字左右的短文，语言流畅有深度，适合直接发公众号",
  "tags": ["标签1", "标签2", "标签3", "标签4", "标签5"],
  "outro_text": "关注 AI不够酷｜获取更多AI资讯"
}
```

**新增字段说明：**

| 字段 | 限制 | 说明 |
|------|------|------|
| `short_article` | 约300字 | 完整的短文章，语言流畅有深度，适合直接发公众号 |
| `tags` | 5-8个 | 用**英文逗号分隔**的标签列表，如 `AI, 大模型, OpenAI, GPT` |

## 核心逻辑：新闻缩写 → 分三段 → 扩展短文 → 打标签

**第一步：用新闻报道口吻缩写为约150字**
将输入文章用**新闻报道的语气**压缩为约150字，要求：
- 像新闻记者写的简讯，简洁专业
- **语言流畅**，读起来顺畅不生硬
- 覆盖全部关键信息（时间、人物、事件、数字、结果）
- 不要求有逻辑递进，只要信息完整、不遗漏关键点即可

**第二步：按字数分三段**
将150字新闻缩写按自然语义分成三段：
- 第一段：40-50字
- 第二段：40-50字
- 第三段：40-50字

**第三步：扩展为300字短文**
以第二步的新闻缩写为基础，用更完整、流畅的语言扩展成约300字的公众号短文。语言要有深度和可读性，适合直接发布。

**第四步：提取5-8个标签**
从文章内容中提取最核心的标签，英文逗号分隔。优先选择：具体产品名、技术方向、行业事件、热门话题。
## 提炼规则

| 字段 | 限制 | 说明 |
|------|------|------|
| `main_title` | ≤10 字 | 一句话概括核心，数字优先 |
| `sub_title` | ≤12 字 | 补充悬念或背景，含数字更佳 |
| `text_sections` | 三段共约150字 | 新闻口吻缩写后按字数分三段 |
| `short_article` | 约300字 | 完整短文，语言流畅有深度，可直接发公众号 |
| `tags` | 5-8个 | 英文逗号分隔的标签，如 `AI, 大模型, OpenAI, GPT` |
| `outro_text` | 默认固定 | "关注 AI不够酷｜获取更多AI资讯" |

## 示例

## 关键原则

1. **新闻口吻**：缩写时像新闻记者写简讯，简洁专业、语言流畅
2. **覆盖关键**：时间、人物、事件、数字、结果都要提到
3. **分句即可**：150字按自然语义或字数切分成三段，不强制递进逻辑
4. **main_title数字优先**："6小时破亿"比"OpenAI发布"更有信息量
5. **输出纯JSON**：不要解释文字，直接给JSON
