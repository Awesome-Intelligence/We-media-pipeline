## 一、文章类型分类（5大类）

| 大类 | 子类型 | 代表文章 | 核心结构 |
|------|--------|---------|---------|
| **产品类** | 测评/发布/对比评测/深度体验 | 2,3,7,11 | 体验/踩坑 → 功能介绍 → 对比分析 → 推荐 |
| **行业研究类** | 趋势/报告/观点/盘点 | 1,4,5,8,12 | 现象/观点 → 分析解读 → 趋势判断 |
| **教程类** | 安装指南/使用教程 | 6 | 场景 → 痛点 → 步骤 → 结果 |
| **人物故事类** | 专访/真实故事 | 9 | 人物 → 反差 → 故事 → 价值 |
| **技术科普类** | 概念解读 | 10 | 概念 → 组件 → 对比 → 总结 |

## 四、段落节奏特点

### 4.1 段落长度
- **超短段**：1句话，金句单独成段
- **短段落**：2-3行，背景/过渡
- **清单段**：第一/第二/第三
- **代码段**：等宽字体，命令行
- **引用段**：「」包裹，单独成段

### 4.2 重点加粗规律

| 位置 | 加粗内容 | 目的 |
|------|---------|------|
| 开篇 | **Karpathy 何许人也？** | 引起注意 |
| 核心观点 | **编程的 AGI 拐点已经到来。** | 金句强调 |
| 小标题 | Coding AGI：2025 年 12 月的拐点 | 结构清晰 |
| 新概念 | **智能体原生经济** | 概念突出 |
| 关键数据 | **40 亿美元** | 数字强调 |
| 步骤标题 | **第一步，安装 nvm** | 步骤引导 |

## 九、配图使用规范

### 9.1 配图数量（按文章类型）

| 文章类型 | 配图数量 | 配图特点 |
|---------|---------|---------|
| **产品测评** | 3-5张 | 产品界面截图、功能演示图 |
| **产品发布** | 2-4张 | 产品界面、功能亮点图 |
| **行业研究** | 4-8张 | 数据图表、趋势图、对比图 |
| **教程类** | 5-10张 | 步骤截图、命令行截图、结果图 |
| **人物故事** | 1-3张 | 人物照片、场景图 |
| **盘点类** | 6-12张 | 各产品Logo、界面截图 |
| **技术科普** | 4-8张 | 架构图、流程图、对比图、总结图 |

### 9.2 配图位置规律

| 位置 | 配图类型 | 作用 |
|------|---------|------|
| **文章顶部** | 封面图 | 吸引点击，视觉引导 |
| **小标题后** | 概念引入图 | 可视化新概念 |
| **段落中间** | 解释说明图 | 辅助理解复杂内容 |
| **章节结尾** | 总结归纳图 | 脑图/总结表格 |

### 9.3 配图频率
- **标准**：每300-500字配1张图
- **上限**：单篇文章不超过12张（避免加载慢）
- **下限**：至少1张封面图

### 9.4 配图样式
- **尺寸**：宽度100%，自适应手机屏幕
- **边框**：圆角4px，与文字有间距
- **说明**：图片下方可加简短说明文字

### 9.5 配图与文字搭配模式

```
模式1：概念解释
【文字】介绍新概念
【配图】架构图/示意图
【文字】详细展开

模式2：对比分析
【文字】提出对比
【配图】对比表格/流程图
【文字】深入解读

模式3：步骤教程
【文字】步骤说明
【配图】操作截图
【文字】注意事项

模式4：总结归纳
【文字】总结要点
【配图】脑图/总结图
【文字】引出下文
```

## 十、关键发现

1. 标题必有「：」或「？」，制造悬念
2. 开篇必用故事/数据/提问，拒绝平铺直叙
3. 段落极短，适合手机阅读
4. 金句单独成段，加粗强调
5. 数据具体，不用"很多"而用"十几款"
6. 口语化但不失专业度
7. 每篇必有"我"的视角，增强代入感

## 十一、配图描述规范

### 11.1 描述原则

配图描述应该**可搜索**。

### 11.2 推荐描述格式

```
[[IMG: 主题词 + 场景/风格]]
```

### 11.3 按文章类型的配图关键词

**产品测评类**：
- `Mobile phone app interface`
- `Person using smartphone`
- `Technology product review`
- `Digital device in hand`

**行业研究类**：
- `Data visualization dashboard`
- `Business chart and graph`
- `Technology trend concept`
- `Futuristic cityscape`

**教程类**：
- `Computer screen coding`
- `Person working on laptop`
- `Step by step tutorial`
- `Technology workspace`

**人物故事类**：
- `Professional portrait`
- `Office worker thinking`
- `Creative workspace`
- `Person with laptop`

**技术科普类**：
- `Abstract technology background`
- `Network connections digital`
- `Circuit board close up`
- `Digital transformation concept`

**盘点类**：
- `Collection of technology icons`
- `Multiple devices on desk`
- `Technology comparison`
- `Grid of app icons`

### 11.4 配图描述模板

```
# 封面图（必配）
[[IMG: 文章主题相关的高质量概念图]]

# 章节配图（选配）
[[IMG: 该章节核心概念的通用场景图]]

# 对比配图（选配）
[[IMG: 对比双方的通用代表图]]

# 总结配图（选配）
[[IMG: 总结性概念的抽象图或示意图]]
```

## 十二、模仿要点

- 用具体数字代替模糊描述
- 用场景代替抽象概念
- 用对话代替陈述
- 用问题引导读者思考
- 用金句收尾引发共鸣
- **配图描述用通用关键词，方便后续搜索**
